# 칼로FC 디자인 시스템

## 철학

**"절제된 밀도"** — 최소한의 시각 요소로 최대한의 정보를 전달한다.

1. **장식 0**: 그림자, 그라데이션, 패턴은 근거 없이 사용하지 않는다
2. **크기가 위계다**: 색이나 장식이 아닌 크기 차이로 정보의 중요도를 표현한다
3. **여백이 구분이다**: 요소 간 구분은 선이나 카드가 아니라 간격으로 한다
4. **숫자가 주인공이다**: 스탯 화면에서 숫자는 가장 크고 라벨은 보조적이다
5. **하나의 focal point**: 한 화면에서 눈이 먼저 가는 곳은 하나다

## 1. 색상 체계

### 1-1. 원칙

- 메인 컬러 3개 이하: `textPrimary`(강조) + `textTertiary`(보조) + `primary`(악센트)
- 경고/상태 색은 결과 표시에만 사용 (W/D/L 도트, MOM 배지)
- 카드/컨테이너 배경은 `surfaceLight` — 흰 배경과 미세한 대비만
- 하드코딩 금지 → `AppColors` 토큰 사용

### 1-2. 토큰

| 역할 | 토큰 | HEX | 용도 |
|------|------|-----|------|
| 강조 텍스트 | `textPrimary` | #333D4B | 이름, 숫자, 제목 |
| 보조 텍스트 | `textSecondary` | #6B7684 | 팀명, 라벨 |
| 비활성 텍스트 | `textTertiary` | #8E97A3 | 날짜, 힌트, 캡션 |
| 브랜드 | `primary` | #1572D1 | 참가하기 CTA, 브랜드 히어로 |
| 카드 배경 | `surfaceLight` | #F6F7F9 | 입력 필드, 카운터, 카드 |
| 페이지 배경 | `surface` | #F2F4F6 | 리스트 배경 |
| 구분선 | `textPrimary` alpha 6% | — | 리스트 행 사이 |
| 비활성 | `iconInactive` | #D1D6DB | 빈 슬롯, 비활성 버튼, 핸들바 |

### 1-3. 성취/MOM 배지 색

| 배지 | 배경 | 텍스트 | 규칙 |
|------|------|--------|------|
| MOM | `momBackground` (#FFF8E1) | `momText` (#F57F17) | AppColors 토큰 |
| 출석왕 | #FFF3E0 | #E65100 | Material 50 → 800~900 |
| 득점 관련 | #E3F2FD | #1565C0 | 〃 |
| 어시스트 | #E8F5E9 | #2E7D32 | 〃 |
| 기타 | #FCE4EC | #C62828 | 〃 |

**규칙**: 배경 = Material 50톤, 텍스트 = 같은 색 800~900톤

### 1-4. 다크 영역 (프로필 히어로 등)

white alpha 3단계로만 위계 표현:

| 역할 | 값 |
|------|---|
| 강조 | white 100% |
| 보조 | white 50% (`0x80FFFFFF`) |
| 미약 | white 15% (`0x26FFFFFF`) |

## 2. 타이포그래피

### 2-1. 폰트 역할

| 폰트 | 용도 |
|------|------|
| **Pretendard** | 모든 한글 텍스트, 본문, 라벨, 버튼 |
| **Barlow Condensed** | 큰 숫자 (스코어, 스탯, 경기 시간) |
| **SCDream** | 팀명 강조 (홈 헤더 등 제한적) |

### 2-2. 숫자 크기 계층 (Barlow Condensed)

스탯/스코어 화면에서 숫자 크기로 위계를 만든다:

| 계층 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| Hero | 44~48pt | w800 | 메인 스코어, 프로필 카드 스탯 |
| Sub | 28~32pt | w800 | 보조 스탯, H2H 전적 숫자 |
| Micro | 20~22pt | w700~w900 | 카드 내 타일 숫자 |

### 2-3. 한글 텍스트 계층 (Pretendard)

| 토큰 | 크기/굵기 | 용도 |
|------|-----------|------|
| 커스텀 | 28/w900 | 프로필 이름 |
| `sectionTitle` | 20/w700 | 섹션 제목 |
| `heading` | 16/w700 | 중간 제목, 섹션 타이틀 |
| `body` | 15/w500 | 본문 기본 |
| `label` | 14/w700 | 라벨 굵은 |
| `labelMedium` | 14/w600 | 라벨 중간 |
| `caption` | 12/w400 | 캡션, 보조 정보 |
| `buttonPrimary` | 17/w700 | 메인 CTA |
| `buttonSecondary` | 16/w600 | 보조 버튼 |

## 3. 여백 시스템 (8px 그리드)

`AppSpacing` 토큰만 사용. 하드코딩 금지 (컴포넌트 고유값만 예외).

| 토큰 | 값 | 용도 |
|------|----|------|
| `xs` | 4 | 아이콘↔텍스트, 같은 블록 내 |
| `sm` | 8 | 기본 소 간격, 요소 간 |
| `md` | 12 | 행 간 갭, 아이콘↔텍스트 (큰) |
| `base` | 16 | 기본 간격, 카드 내부 패딩 |
| `lg` | 20 | **페이지 좌우 패딩** |
| `xl` | 24 | 섹션 패딩 |
| `xxl` | 32 | 섹션 간 간격 |
| `xxxl` | 48 | 대맥락 전환 |

### 여백 리듬 (일관성 규칙)

| 맥락 | 여백 |
|------|------|
| 같은 블록 내 | xs~sm (4~8) |
| 요소 간 | md~base (12~16) |
| 섹션 간 | xl~xxl (24~32) |
| 대맥락 전환 | xxxl (48) |

## 4. 라운딩 (Squircle)

`SmoothBorderRadius` + `cornerSmoothing: 1.0`. 캐시 인스턴스만 사용.

| 토큰 | 값 | 용도 |
|------|-----|------|
| `smoothXs` | 4 | 팀 로고 클립, 작은 뱃지 |
| `smoothSm` | 8 | 매치 카드, 캡슐 |
| `smoothMd` | 12 | 버튼, 컨테이너, 입력 필드 |
| `smoothButton` | 14 | CTA 버튼 |
| `smoothLg` | 16 | 큰 카드 |
| `smoothXl` | 20 | 바텀시트 상단 |
| `smoothFull` | 100 | 캡슐/원형 pill |

## 5. 컴포넌트 패턴

### 5-1. 선택 요소 (Outline pill)

단일/복수 선택 모두 동일한 outline pill 사용:

```
비선택: surfaceLight 배경 + 투명 테두리 + textTertiary 텍스트
선택:   surfaceLight 배경 + textPrimary 1.5px 테두리 + textPrimary 텍스트
```

- 배경색 불변 → 여러 개 선택해도 산만하지 않음
- pill (`smoothFull`) = 필터/태그 역할
- 둥근 네모 (`smoothSm`) = 버튼/그리드 역할
- **금지**: filled 파란 배경 (CTA 버튼과 혼동)

사용처: 참가하기 모달 포지션/쿼터, 프로필 편집 포지션/주발

### 5-2. 카운터 (± pill)

```
┌─────────────┐
│  −    0    + │  surfaceLight, smoothFull, 72×34
└─────────────┘
```

- 값 0: textTertiary 숫자, iconInactive 마이너스
- 값 >0: textPrimary 숫자
- 터치 시 lightImpact 햅틱

사용처: 경기 결과 입력 (골/도움)

### 5-3. 입력 필드

```
surfaceLight 배경 + smoothSm 라운드 + border 없음
TextField: body 스타일, textPrimary 색
```

사용처: 프로필 편집 (이름, 등번호, 키)

### 5-4. CTA 버튼

| 종류 | 배경 | 텍스트 | 용도 |
|------|------|--------|------|
| Primary | `textPrimary` (검정) | white | 저장, 확인 |
| Brand | `primary` (파랑) | white | 참가하기 |
| Secondary | `surface` (회색) | textPrimary | 보조 액션 |

모두 `elevation: 0` + `smoothButton` 라운드.

### 5-5. 바텀시트

```
흰 배경 + squircle 상단 (xl) + 드래그 핸들 (36×4, iconInactive)
제목: heading + textPrimary
섹션 라벨: captionMedium + textSecondary + w800
```

### 5-6. 경기 기록 행 (카드형)

```
┌────────────────────────────────────┐
│ [로고36] FC쏘아    MOM  │ 1골 1도움 ││  surfaceLight
│          3/14           │ white bg  ││  smoothMd
└────────────────────────────────────┘
```

행 간격 sm(8) — divider 대신 간격으로 구분.

### 5-7. H2H 전적 Summary

```
     3           1           1        ← 32pt w900
     승          무           패
  ████████████░░░░░████████           ← 6px 비율 바
  FC칼로      최근 5경기     FC쏘아
```

승 = `primary`, 무 = `iconInactive`, 패 = `textPrimary`

### 5-8. 이벤트 카드 (지연 등장)

경기 종료 후 결과 입력 같은 이벤트성 카드:

- **400ms 딜레이** 후 등장 (영구 콘텐츠와 구분)
- **아래→위 슬라이드 + 페이드** (350ms, easeOutCubic)
- X 버튼으로 dismiss
- `surfaceLight` 배경

## 6. 화면별 패턴

### 6-1. 프로필 (공유 가능한 카드)

```
surfaceLight 카드:
  이름 (Pretendard 28/w900)
  팀 · 포지션 · 번호 · 시즌 (caption)
  상반신 이미지 (가운데, 크게)
  4열 스탯 (Barlow 44pt)
  태그 pill 행 (다채로운 색상)
```

카드 밖: 피지컬 정보 (테두리 행) → 최근 기록 (카드형 행)

### 6-2. 라인업 빌더

```
피치 (Expanded, 배경 #4F7E67):
  좌상단: 공정성 오버레이 (흰 pill)
  우상단: 포메이션 드롭다운 (흰 pill)
  슬롯: 원형 42px + 이름 13pt white
  빈 슬롯: + 아이콘, 탭 → 멤버 피커 시트
  
벤치 (하단 가로 스크롤):
  원형 44px 아바타 + 이름
  
쿼터: 슬라이딩 세그먼트 컨트롤 (surface 배경 + 흰 인디케이터)
```

### 6-3. 결과 입력

```
상단: 스코어 (로고36 + Barlow 48pt + ± surfaceLight 원형 버튼)
구분선
하단: 개인 기록 (아바타36 원형 + 이름 + 카운터 2개)
고정 하단: 저장 CTA (검정)
```

한 화면에 전부 (스텝 없음). 심플.

### 6-4. 참가하기 모달

```
참가하기
선호 포지션: outline pill Wrap (GK LB CB RB DM CM AM LW ST RW)
참가 가능 쿼터: outline pill Wrap (Q1 Q2 Q3 Q4)
참가 완료 CTA (파랑)
```

포지션/쿼터 모두 같은 pill 스타일. 서브타이틀 없음. CTA 라벨 고정.

## 7. 금지 사항

1. 색상/여백/라운드/텍스트 **하드코딩 금지** → 토큰 사용
2. **그림자 금지** → 배경색 차이로 카드 구분
3. **filled 색상 버튼 남용 금지** → CTA만 filled, 나머지는 outline/surface
4. **프로그레스바 금지** → 비율은 숫자로 표현 (83% > 프로그레스바)
5. **이모지 아이콘 금지** → Material Icons 사용
6. **3색 이상 멀티컬러 금지** → 성취 배지 제외
7. **카드 안 카드 금지** → 중첩 1레벨까지
8. **장식적 패턴/그라데이션 금지** → 정보 미전달 장식 제거

## 8. 새 화면 만들 때 체크리스트

### 필수
- [ ] 좌우 패딩 `lg` (20)?
- [ ] 섹션 간 갭 `xxl` (32)?
- [ ] 모든 색상이 `AppColors` 토큰?
- [ ] 모든 텍스트가 `AppTextStyles` 토큰? (`.copyWith(color:)` 패턴)
- [ ] 모든 라운딩이 `AppRadius.smooth*` 캐시 인스턴스?
- [ ] 그림자 사용 안 함?

### 위계
- [ ] 한 화면에 메인 컬러 3개 이하?
- [ ] 카드 안에 카드 없음? (1레벨까지)

### 인터랙션
- [ ] 탭 영역 ≥ 44×44pt?
- [ ] 중요 토글에 햅틱 (mediumImpact)?
- [ ] 선택/이동에 햅틱 (selectionClick/lightImpact)?
