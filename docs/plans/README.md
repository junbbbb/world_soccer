# Plans

`/build` 실행 시 대규모 작업의 단계별 진행 상황을 기록하는 디렉토리.

## 파일 규칙

- 파일명: `YYYYMMDD-작업명.md` (예: `20260412-supabase-연동.md`)
- 완료된 플랜은 삭제하지 않고 보관 (히스토리)
- 각 플랜은 아래 템플릿 사용

## 템플릿

```markdown
# [작업명]

시작: YYYY-MM-DD
상태: 진행중 | 완료 | 보류

## 단계

- [x] 단계 1 — 설명
- [ ] 단계 2 — 설명
- [ ] 단계 3 — 설명

## 메모

변경 사항, 결정, 이슈 기록.
```
